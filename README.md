## 2. 執行

```bash
# 每日盤後選股（用內建精選股票池，台股約 15:30 後資料才完整）
python tw_momentum_screener.py

# 掃描「全上市櫃」：自動抓全市場，依價/量預篩，取流動性最高前 250 檔

# 臨時調整信心閥值（例如盤勢好時放寬到 65）

# 關閉新聞情緒抓取（速度更快 / 無網路時）
python tw_momentum_screener.py --no-sentiment

# 把結果匯出成 CSV（方便手機留存 / 分享，Excel 開不亂碼）
python tw_momentum_screener.py --csv

# 族群（產業）強度排行榜 —— 看資金流向哪個族群

# 族群排行榜 + 視覺化圖表（族群強度橫條圖 + 輪動趨勢線，需 matplotlib）
python tw_momentum_screener.py --groups --chart
```

### 參數說明
| 旗標 | 作用 |
|------|------|
| `--full` | 掃描全上市櫃（先用 OpenAPI 預篩，僅對流動性最高的前 `max_universe` 檔抓歷史，兼顧覆蓋率與手機效能） |
| `--threshold N` | 覆寫信心閥值 |
| `--backtest --days N` | 回測最近 N 個交易日 |
| `--no-sentiment` | 跳過新聞情緒抓取 |
| `--no-industry` | 關閉族群（產業）相對強度分析 |
| `--finmind` | 回測時用 FinMind 抓歷史三大法人，讓回測**納入籌碼面** |
| `--csv [檔名]` | 匯出選股結果為 CSV |
| `--groups` | 只輸出族群（產業）強度排行榜（純文字、零相依） |
| `--chart` | 搭配 `--groups`：輸出族群強度橫條圖與輪動趨勢圖（需 `matplotlib`） |
| `--no-groups` | 每日選股時**不**顯示族群排行榜（預設會先顯示族群脈絡再給選股） |

> 💡 **每日流程**：直接 `python tw_momentum_screener.py` 會先印出**族群強度排行榜**
> （看今天資金集中在哪些族群），接著才是符合標準的飆股清單與精選 Top 3 ——
> 「先看族群、再看個股」。不想看族群可加 `--no-groups`。

### 族群輪動分析 🔄
資金會在產業族群間輪流流動，而飆股多成群出現。本工具提供：
- **族群強度排行榜**（`--groups`）：把各產業依「族群相對強度」由強到弱排序，
  顯示檔數與帶頭股，純文字、零相依，每天必看。
- **視覺化**（`--chart`，需 `pip install matplotlib`）：
  - `group_strength.png`：族群強度橫條圖（當日快照）
  - `group_rotation.png`：近 N 日各族群強度的趨勢線，一眼看出誰在升溫、誰在退燒。
  - 圖表會自動套用系統的中文字型（iPhone 內建 PingFang，桌機可裝 Noto Sans CJK）。

### 上市 + 上櫃全涵蓋 🔵
- **內建股票池**同時含上市（`.TW`）與上櫃（`.TWO`）熱門股。
- **`--full`** 會自動抓 **TWSE（上市）+ TPEx（上櫃）** 全市場清單再預篩
  （已排除 `00` 開頭的 ETF/ETN，只留個股；任一市場抓取失敗會明確告警）。
- **三大法人籌碼**同時整合證交所與櫃買中心，上櫃股也有籌碼分數。
- 報告會標示每檔的**市場別**，並顯示「上市 X / 上櫃 Y」的分布。

### 風控出場建議（ATR）
每檔精選 Top 都附 **ATR 動態停損停利**：停損 = 收盤 − 2×ATR，停利 = 收盤 + 3×ATR
（風報比約 1:1.5），協助你在進場前先想好出場。

## 3. 程式邏輯解說

### 3.1 為什麼是「這幾個指標」？— 台股抓飆股的共識

研究歐奈爾 CANSLIM、李佛摩關鍵點，以及台股實務派（如籌碼面選股）的共同特徵，
飆股啟動前幾乎都具備：

1. **高相對強度 (RS)**：股價表現「贏過大盤、贏過多數個股」是飆股最重要的單一特徵。
2. **創新高 + 帶量突破**：李佛摩的「關鍵點」與歐奈爾的「杯柄突破」本質都是 —
   股價突破整理區、創新高，且**成交量明顯放大**（買盤認真）。
3. **均線多頭排列**：價 > 20MA > 60MA > 120MA，確認中長期趨勢向上。
4. **動能確認**：MACD 柱狀體翻正轉強、RSI 落在 55~80 的「強勢但未過熱」甜蜜帶。
5. **籌碼集中**：三大法人（尤其外資、投信）買超，代表大資金進場。
6. **題材/情緒**：新聞與總經順風，能延長噴出時間。

### 3.2 評分權重（總和 = 100%）

| 子項 | 權重 | 為什麼是這個權重 |
|------|------|----------------|
| 個股相對強度 RS | **15%** | 歐奈爾實證最有效的因子；與族群 RS 合計 25% |
| 族群（產業）相對強度 | **10%** | IBD「產業群組排名」概念：飆股多成群出現，強勢族群帶動帶頭股 |
| 突破力道（創新高+爆量） | **25%** | 飆股的「點火」訊號，核心因子 |
| 均線多頭排列 | **20%** | 趨勢確認，避免抓到反彈而非主升段 |
| MACD/RSI 動能 | **15%** | 二次確認，過濾假突破；RSI>80 反而扣分避免追最末段 |
| 三大法人籌碼 | **10%** | 大資金背書，但台股法人資料有延遲，故權重適中 |
| 新聞/情緒 | **5%** | 加分項，預設中性；資料品質不穩故權重最低 |

> **族群相對強度**：依產業別分組，族群強度 = 該族群成員個股 RS 百分位平均；
> 個股的族群分數 = 所屬族群強度。族群中 RS 最高者（且族群夠強）標記為 **👑族群帶頭股**。
> 資料來自 TWSE/TPEx 公司基本資料 OpenAPI；無資料時自動退化為個股 RS（不影響運作）。

> RS 分數採**全市場百分位排名**：把當日通過篩選的個股依多週期超額報酬排序，
> 轉成 0~100 的百分位（接近 IBD RS Rating 的概念），確保是「相對」強度而非絕對漲幅。

### 3.3 「寧缺勿濫」如何實作？— 三道關卡

1. **硬性過濾（直接淘汰）**
   - 股價 < 10 元（排除水餃股）
   - 20 日均量 < 100 萬股（流動性不足）
   - 收盤未站上 52 週高點的 90%（離高點太遠，不是主升段候選）

2. **信心閥值（預設 75 分，嚴格版）**
   - 加權總分未達 `confidence_threshold` 的標的**一律不輸出**。
   - 盤勢差時，通過者自然變少 —— 程式會**如實輸出 5 檔、甚至 0 檔**，
     並印出「寧可空手」的提示，**絕不為了湊滿名額而降低標準**。

3. **數量上限（最多 10 檔）+ 精選 Top 3**
   - 通過閥值者按信心分數排序，取前 10。
   - 再從中挑分數最高的 3 檔作為首選推薦（合格不足 3 檔時自動少給），並列出各子分數供你判斷。

### 3.4 回測框架

- **走勢回放**：對最近 N 個交易日，每天只用「截至當日」的價量產生訊號（避免未來函數），
  以**隔日開盤價**買入、持有 `bt_hold_days`（預設 5 日）後的收盤價賣出計算報酬。
- **輸出**：訊號數、達標勝率（5 日 +3% 視為成功）、平均/中位數報酬、最佳最差。
- **籌碼面回放**：加上 `--finmind` 後，回測會用 FinMind 抓**歷史三大法人**，
  讓每個訊號日的籌碼分數使用「當日真實買賣超」而非中性值，使回測更貼近實戰。
- 回測未計入交易成本與滑價；未加 `--finmind` 時籌碼用中性值（結果偏保守）。
