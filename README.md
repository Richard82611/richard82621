## 2. 執行

```bash
# 每日盤後選股（用內建精選股票池，台股約 15:30 後資料才完整）
python tw_momentum_screener.py

# 掃描「全上市櫃」：自動抓全市場，依價/量預篩，取流動性最高前 250 檔

# 歷史回測（驗證策略勝率）

# 臨時調整信心閥值（例如盤勢好時放寬到 65）
```

### 參數說明
| 旗標 | 作用 |
|------|------|
| `--full` | 掃描全上市櫃（先用 OpenAPI 預篩，僅對流動性最高的前 `max_universe` 檔抓歷史，兼顧覆蓋率與手機效能） |
| `--threshold N` | 覆寫信心閥值 |
| `--backtest --days N` | 回測最近 N 個交易日 |
| `--no-sentiment` | 跳過新聞情緒抓取 |
| `--csv [檔名]` | 匯出選股結果為 CSV |

### 上市 + 上櫃全涵蓋 🔵
- **內建股票池**同時含上市（`.TW`）與上櫃（`.TWO`）熱門股。
- **`--full`** 會自動抓 **TWSE（上市）+ TPEx（上櫃）** 全市場清單再預篩。
- **三大法人籌碼**同時整合證交所與櫃買中心，上櫃股也有籌碼分數。
- 報告會標示每檔的**市場別**，並顯示「上市 X / 上櫃 Y」的分布。

### 風控出場建議（ATR）
每檔精選 Top 都附 **ATR 動態停損停利**：停損 = 收盤 − 2×ATR，停利 = 收盤 + 3×ATR
（風報比約 1:1.5），協助你在進場前先想好出場。

## 3. 程式邏輯解說

### 3.1 為什麼是「這幾個指標」？— 台股抓飆股的共識

研究歐奈爾 CANSLIM、李佛摩關鍵點，以及台股實務派（如籌碼面選股）的共同特徵，
飆股啟動前幾乎都具備：

1. **高相對強度 (RS)**：股價表現「贏過大盤、贏過多數個股」是飆股最重要的單一特徵。
2. **創新高 + 帶量突破**：李佛摩的「關鍵點」與歐奈爾的「杯柄突破」本質都是 —
   股價突破整理區、創新高，且**成交量明顯放大**（買盤認真）。
3. **均線多頭排列**：價 > 20MA > 60MA > 120MA，確認中長期趨勢向上。
4. **動能確認**：MACD 柱狀體翻正轉強、RSI 落在 55~80 的「強勢但未過熱」甜蜜帶。
5. **籌碼集中**：三大法人（尤其外資、投信）買超，代表大資金進場。
6. **題材/情緒**：新聞與總經順風，能延長噴出時間。

### 3.2 評分權重（總和 = 100%）

| 子項 | 權重 | 為什麼是這個權重 |
|------|------|----------------|
| 相對強度 RS | **25%** | 歐奈爾實證最有效的單一因子，給最高權重 |
| 突破力道（創新高+爆量） | **25%** | 飆股的「點火」訊號，與 RS 並列核心 |
| 均線多頭排列 | **20%** | 趨勢確認，避免抓到反彈而非主升段 |
| MACD/RSI 動能 | **15%** | 二次確認，過濾假突破；RSI>80 反而扣分避免追最末段 |
| 三大法人籌碼 | **10%** | 大資金背書，但台股法人資料有延遲，故權重適中 |
| 新聞/情緒 | **5%** | 加分項，預設中性；資料品質不穩故權重最低 |

> RS 分數採**全市場百分位排名**：把當日通過篩選的個股依多週期超額報酬排序，
> 轉成 0~100 的百分位（接近 IBD RS Rating 的概念），確保是「相對」強度而非絕對漲幅。

### 3.3 「寧缺勿濫」如何實作？— 三道關卡

1. **硬性過濾（直接淘汰）**
   - 股價 < 10 元（排除水餃股）
   - 20 日均量 < 100 萬股（流動性不足）
   - 收盤未站上 52 週高點的 90%（離高點太遠，不是主升段候選）

2. **信心閥值（預設 70 分）**
   - 加權總分未達 `confidence_threshold` 的標的**一律不輸出**。
   - 盤勢差時，通過者自然變少 —— 程式會**如實輸出 5 檔、甚至 0 檔**，
     並印出「寧可空手」的提示，**絕不為了湊滿 20 檔而降低標準**。

3. **數量上限（最多 20 檔）+ 精選 Top 3**
   - 通過閥值者按信心分數排序，取前 20。
   - 再從中挑分數最高的 3 檔作為首選推薦，並列出各子分數供你判斷。

### 3.4 回測框架

- **走勢回放**：對最近 N 個交易日，每天只用「截至當日」的價量產生訊號（避免未來函數），
  檢查訊號後持有 `bt_hold_days`（預設 5 日）的報酬。
- **輸出**：訊號數、達標勝率（5 日 +3% 視為成功）、平均/中位數報酬、最佳最差。
- 回測為求保守，**未計入**交易成本、滑價，也**未回放歷史籌碼/新聞**（用中性值），
  故實際整合籌碼後的表現通常會更好。

## 4. 資料來源與 API 建議（皆免金鑰 / 有免費額度）

| 資料 | 來源 | 狀態 |
|------|------|------|
| 盤後股價（歷史） | **yfinance**（Yahoo Finance） | ✅ 已內建，免金鑰；代碼用 `2330.TW`/`6488.TWO` |
| 全市場當日清單 | **TWSE / TPEx OpenAPI** | ✅ 已內建（`--full` 用），免金鑰，單一請求拿全市場 |
| 三大法人買賣超（上市） | **證交所 TWSE**（T86 報表） | ✅ 已內建，免金鑰；盤後約 15:30 更新 |
| 三大法人買賣超（上櫃） | **櫃買中心 OpenAPI** | ✅ 已內建（`_fetch_tpex_netbuy()`） |
| 新聞情緒 | **Google News RSS + 關鍵字詞典** | ✅ 已內建，免金鑰；只對前段班 shortlist 抓取省流量 |
| 更完整台股資料 | **FinMind**（finmind.github.io） | 選用，免費註冊有額度，含法人、融資券、股利等 |

> **新聞情緒運作方式**：對信心分數前 `sentiment_shortlist`（預設 30）檔，
> 抓 Google News RSS 近期標題 → 用正/負面關鍵字詞典計分 → 映射到 -1~+1 →
> 納入 5% 權重並標註「新聞偏多/偏空」。詞典可在 `_POS_WORDS` / `_NEG_WORDS` 自行擴充。
